# 基于 AI 粒子群算法的线控转向 PID 自动调参系统

## 项目简介

这是一个基于 **粒子群优化算法 (PSO)** 的汽车线控转向系统 PID 参数自动调优工具。通过 AI 算法自动寻找最优的 PID 控制参数,实现快速、无超调、高精度的转向控制。

## 功能特性

- **物理模型仿真**: 使用二阶传递函数模拟线控转向执行电机系统
- **手动调参模式**: 通过滑块实时调整 P、I、D 参数,观察响应曲线变化
- **AI 自动寻优**: 采用粒子群算法自动搜索最优 PID 参数
- **性能指标对比**: 实时显示超调量、调节时间、稳态误差等关键指标
- **可视化展示**:
  - 阶跃响应对比曲线
  - PSO 算法收敛曲线
  - 交互式图表 (基于 Plotly)

## 技术栈

- **Python 3.x**

## 使用说明

### 手动调参模式

1. 在左侧边栏选择 "手动调参"
2. 使用滑块调整 P、I、D 参数
3. 右侧主界面实时显示阶跃响应曲线和性能指标

### AI 自动寻优模式

1. 在左侧边栏选择 "AI 自动寻优"
2. 设置 PSO 算法参数:
   - **粒子数量**: 建议 20-30 (数量越多搜索越全面,但耗时更长)
   - **迭代次数**: 建议 30-50 (次数越多收敛越好)
   - **目标转角**: 设置阶跃信号的目标值 (默认 30°)
3. 点击 "🚀 启动 AI 自动寻优" 按钮
4. 等待算法完成后,查看优化结果和收敛曲线

### 性能指标说明

- **超调量 (%)**: 响应峰值超过目标值的百分比,越小越好
- **调节时间 (s)**: 系统稳定在目标值 ±2% 范围内所需时间,越短越好
- **稳态误差 (°)**: 稳定后与目标值的偏差,越小越好

## 系统架构

### 1. 物理模型

使用二阶传递函数模拟线控转向系统:

```
G(s) = 100 / (s² + 10s + 100)
```

这是一个典型的欠阻尼二阶系统,适合演示 PID 控制效果。

### 2. PSO 粒子群算法

- **优化维度**: 3 维 (P, I, D)
- **搜索空间**: P ∈ [0, 50], I ∈ [0, 10], D ∈ [0, 10]
- **适应度函数**: ITAE + 超调量惩罚 + 调节时间惩罚

### 3. PID 控制器

采用标准 PID 控制算法:

```
u(t) = Kp·e(t) + Ki·∫e(t)dt + Kd·de(t)/dt
```

## 核心代码说明

### PSO 类

实现了完整的粒子群优化算法,包括:
- 粒子初始化
- 速度和位置更新
- 个体最佳与全局最佳追踪
- 边界处理

### SteeringSystem 类

封装了线控转向系统的仿真逻辑:
- 二阶系统状态空间表示
- PID 闭环控制仿真
- 性能指标计算 (超调量、调节时间、稳态误差)

### 适应度函数

综合考虑多个性能指标:
```python
fitness = ITAE + 100×超调量 + 50×调节时间 + 10×稳态误差
```

## 故障排除

### 常见问题

**Q1: 运行时出现 "ValueError: setting an array element with a sequence"**

A: 这个问题已在 v1.1.0 版本修复。请确保使用最新版本的 `app.py`。

**Q2: 运行时出现 "AttributeError: module 'numpy' has no attribute 'trapz'"**

A: 这个问题已在 v1.2.0 版本修复。这是 NumPy 2.0+ 的兼容性问题，最新版本已自动兼容新旧版本。

**Q3: 依赖包安装失败**

A: 尝试以下解决方案:
```bash
# 升级 pip
python -m pip install --upgrade pip

# 重新安装依赖
pip install streamlit numpy scipy matplotlib plotly --upgrade
```

**Q4: 应用启动缓慢或 AI 寻优速度慢**

A: 减少粒子数量和迭代次数,例如设置为 15 个粒子、20 次迭代。

**Q5: 图表不显示或显示异常**

A:
- 刷新浏览器 (F5)
- 清除浏览器缓存
- 使用 Chrome 或 Firefox 浏览器
